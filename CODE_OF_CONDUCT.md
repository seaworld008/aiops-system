# Code of Conduct

## Our commitment

We are committed to a welcoming, respectful, and technically rigorous community. Participation must be free from harassment and discrimination, regardless of background, identity, experience level, role, employer, or location.

## Expected behavior

- Be respectful, patient, and specific.
- Critique ideas and code, not people.
- State assumptions and acknowledge uncertainty.
- Welcome questions and help contributors learn the system's safety boundaries.
- Protect private operational, security, and personal information.
- Accept maintainer decisions made to preserve user safety, even when they narrow a feature.

## Unacceptable behavior

- Harassment, threats, insults, sexualized language, or discriminatory conduct.
- Publishing private information without permission.
- Deliberately weakening safety controls, concealing security impact, or encouraging unsafe production use.
- Sustained disruption, bad-faith review, spam, or impersonation.

## Enforcement

For ordinary project conduct concerns, contact the repository owner privately through their GitHub profile. For security-sensitive matters, use the process in [SECURITY.md](SECURITY.md). Maintainers may edit or remove content, limit participation, or ban contributors when necessary to protect the community.

Enforcement decisions should be proportionate, documented privately, and made without retaliation. Project maintainers are responsible for applying these standards consistently.

## Scope

This code applies in repository spaces and when someone is representing the project in public or private community interactions.
